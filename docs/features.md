# Features - OpenClaw Launch

OpenClaw Launch helps teams choose a model, bind a channel, pay for a plan, track provisioning, and operate OpenClaw instances from a hosted console.

## Model and channel selection

Pick the model family and channel target before deployment so the launch matches the audience and support workflow.

## Provisioning visibility

Track order state, deployment state, instance labels, console URLs, and upgrade status in one place.

## Messaging-channel setup

Prepare Telegram, Discord, WhatsApp, or similar channel binding without turning token handling into a separate project.

## Guest and account flow

Let a buyer start with a guest order, then bind access and console state after payment.

## Operational console

Use the console for follow-up deployments, runtime notes, channel state, and support handoff after the first launch.

## Security-aware defaults

Keep credential handling, deployment logs, and access state visible enough for real teams to review.

## Pricing and Support

Plans cover deployment capacity, console access, channel setup, runtime tracking, upgrade help, and support through the public pricing flow.
